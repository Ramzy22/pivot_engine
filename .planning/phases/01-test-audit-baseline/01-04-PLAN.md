<task type="auto">
  <name>Task 1: Fix remaining test failures</name>
  <files>Any test files with failures (determined from audit_raw.txt)</files>
  <action>
    Read `audit_raw.txt` from plan 02. For each FAILED test, determine the root cause and fix it. Apply fixes in this priority order:

    1. **Import errors / missing dependencies** — add skip guards (same pattern as plan 03) if a test requires an external service (ClickHouse, Redis, etc.) that isn't available in the dev environment. Use `pytest.importorskip()` or `@pytest.mark.skipif(condition, reason=...)`.

    2. **AssertionError due to data mismatch** — read the test and the source code it exercises. Fix the test assertion if the test's expected value is wrong, OR fix the source code bug if the test is correct and the source code is wrong. CRITICAL: use the audit output to distinguish these cases — if the test was clearly designed to test a real behavior, the source code is wrong; if the expected value looks like a placeholder, the test is wrong.

    3. **AttributeError / NameError** — if a test calls a method or attribute that was renamed or removed, update the test to call the current API (do not add back deleted code to production).

    4. **Async tests failing with "coroutine never awaited"** — add `@pytest.mark.asyncio` decorator if missing, or add `asyncio_mode = "auto"` to `pivot_engine/pyproject.toml` `[tool.pytest.ini_options]` to avoid per-test decoration.

    For each fix, run the specific test file to confirm it passes before moving on:
    ```
    python -m pytest pivot_engine/tests/test_XXXX.py -v --tb=short
    ```

    After all individual fixes, run the full suite to confirm no regressions:
    ```
    python -m pytest tests/ test_expand_all_backend.py test_filtering.py pivot_engine/tests/ pivot_engine/test_arrow_conversion.py pivot_engine/test_async_changes.py pivot_engine/test_cursor_simple.py pivot_engine/test_scalable_async_changes.py pivot_engine/test_totals_demo.py -v --tb=short 2>&1 | tee .planning/phases/01-test-audit-baseline/final_run.txt
    ```

    The target state: zero FAILED, any number of SKIPPED (skips are acceptable for tests that require unavailable external services like ClickHouse or Redis).

    RULE: Do not modify any file in `pivot_engine/pivot_engine/` (production source) unless a test is clearly correct and the source is the bug. If uncertain, skip the test and note it in TEST_BASELINE.md.
  </action>
