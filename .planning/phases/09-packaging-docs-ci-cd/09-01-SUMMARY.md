# Phase 09 Plan 01: Packaging Baseline Summary

**One-liner:** Canonical pyproject.toml for dash-tanstack-pivot with redis extra, clean npm build contract, and 6-test packaging smoke suite.

## What Was Built

A complete, executable packaging baseline for the `dash-tanstack-pivot` Dash component library:

1. **`dash_tanstack_pivot/pyproject.toml`** — PEP 517/518 canonical metadata declaring `name = "dash-tanstack-pivot"`, `version = "0.0.2"`, `dash>=2.0.0` base dependency, and `optional-dependencies.redis = [redis>=4.0.0]` for `pip install dash-tanstack-pivot[redis]`.

2. **`dash_tanstack_pivot/setup.py`** — Replaced fragmented duplicate dependency/version definitions with a 4-line compatibility shim (`setup()` delegates entirely to pyproject.toml).

3. **`dash_tanstack_pivot/package.json`** — Removed `prepublishOnly` hook and `validate-init` script that referenced a non-existent `_validate_init.py`. npm publish/build is now executable without errors.

4. **`dash_tanstack_pivot/MANIFEST.in`** — Removed `dash_tanstack_pivot.min.js.map` declaration; the source map file is not generated by the current webpack config, causing sdist build warnings.

5. **`dash_tanstack_pivot/dash_tanstack_pivot/__init__.py`** — Added `importlib.metadata` version resolution at top-level with `package-info.json` fallback, making `__version__` canonical post-install while preserving Dash component generator compatibility.

6. **`tests/test_packaging_smoke.py`** — 6 automated smoke tests covering: bundle artifact existence, MANIFEST.in integrity (no declared-but-missing files), pyproject.toml name assertion, redis extra declaration, broken prepublish hook absence, and DashTanstackPivot import surface.
