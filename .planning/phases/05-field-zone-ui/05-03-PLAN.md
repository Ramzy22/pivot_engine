<objective>
Close the Phase 05 sidebar-filter anchoring gap by wiring the existing chip click anchor into the sidebar `FilterPopover` path and hardening the popover so it never falls back to the screen origin.

Purpose: manual verification found that opening a filter from a sidebar filter chip renders the popover in the top-left corner instead of next to the selected chip. The current component already records `filterAnchorEl` in `handleFilterClick`, and the header-level filter popover already consumes it. The sidebar filter-chip popover render uses the same `FilterPopover` component but omits `anchorEl`, so `FilterPopover` cannot resolve a target and leaves its default `{ top: 0, left: 0 }` position.

Output: sidebar filter-chip popovers open next to the clicked chip, stay inside the viewport, and preserve the existing filter state/update flow.
</objective>
