## Directory Purposes

**`dash_tanstack_pivot/src/lib/components/`:**
- Purpose: React component source. The main file `DashTanstackPivot.react.js` is the sole exported Dash component. Sub-directories contain supporting UI components that are imported directly (not lazy-loaded).
- Key files:
  - `DashTanstackPivot.react.js` — everything: table rendering, virtualization, state machine, Dash prop I/O
  - `Filters/FilterPopover.js` — anchored popover for column filters
  - `Sidebar/ColumnTreeItem.js` — drag-and-drop field zone items
  - `Table/EditableCell.js` — inline cell editing with validation
  - `Table/StatusBar.js` — selection count / row count footer

**`dash_tanstack_pivot/src/lib/hooks/`:**
- Purpose: Stateful logic extracted from the main component to keep it maintainable.
- `useServerSideRowModel.js` — the most critical hook; manages the entire server-side data lifecycle
- `useRowCache.js` — the block cache; no rendering logic

**`dash_tanstack_pivot/src/lib/utils/`:**
- Purpose: Pure functions and constants with no React state.
- `styles.js` — four built-in themes (`light`, `dark`, `material`, `balham`) and `getStyles(theme)` factory
- `helpers.js` — `formatValue`, `getKey` (column key builder), `getAllLeafIdsFromColumn`, `isGroupColumn`

**`dash_tanstack_pivot/dash_tanstack_pivot/`:**
- Purpose: The installable Python package. Contents are auto-generated by `npm run build:py` (Dash component generator) plus the Webpack bundle. Do not hand-edit `DashTanstackPivot.py` or `metadata.json` — regenerate via build.

**`pivot_engine/pivot_engine/`:**
- Purpose: The Python OLAP engine. Installable as `scalable-pivot-engine` (see `pivot_engine/pyproject.toml`).
- `tanstack_adapter.py` — the integration seam; the only file that knows about both the TanStack wire format and `PivotSpec`
- `runtime/` — the Dash-specific wiring; agnostic of the engine internals
- `scalable_pivot_controller.py` — the workhorse; do not modify without understanding the materialized hierarchy flow

**`dash_presentation/`:**
- Purpose: A fully-working Dash app that demonstrates and validates the entire stack. Used as the primary integration test harness. `app.py` shows canonical usage of `create_tanstack_adapter`, `PivotRuntimeService`, and the component props.

## Key File Locations

**Entry Points:**
- `dash_presentation/app.py`: Dash application that runs the full stack
- `dash_tanstack_pivot/src/lib/index.js`: JS package entry point

**Configuration:**
- `dash_tanstack_pivot/package.json`: JS dependencies and build scripts
- `pivot_engine/pyproject.toml`: Python package metadata and dependencies
- `dash_tanstack_pivot/webpack.config.js`: Webpack production build config

**Core Logic:**
- `dash_tanstack_pivot/src/lib/components/DashTanstackPivot.react.js`: Main React component
- `dash_tanstack_pivot/src/lib/hooks/useServerSideRowModel.js`: Server-side pagination and cache
- `dash_tanstack_pivot/src/lib/hooks/useRowCache.js`: Block cache implementation
- `pivot_engine/pivot_engine/tanstack_adapter.py`: Python adapter (request/response translation)
- `pivot_engine/pivot_engine/scalable_pivot_controller.py`: OLAP query engine
- `pivot_engine/pivot_engine/hierarchical_scroll_manager.py`: Row window slicing
- `pivot_engine/pivot_engine/runtime/service.py`: `PivotRuntimeService`
- `pivot_engine/pivot_engine/runtime/dash_callbacks.py`: Dash callback registration
- `pivot_engine/pivot_engine/types/pivot_spec.py`: `PivotSpec`, `Measure`, `PivotConfig` types

**Testing:**
- `pivot_engine/tests/` — pytest test suite for backend
- `tests/` — additional root-level tests (some frontend contract tests)

## Naming Conventions

**Files:**
- React components: `PascalCase.react.js` (Dash convention) or `PascalCase.js`
- React hooks: `camelCase` prefixed with `use`, e.g. `useServerSideRowModel.js`
- Python modules: `snake_case.py`
- Sub-components grouped in plural-noun directories: `Filters/`, `Sidebar/`, `Table/`

**Directories:**
- Frontend: `src/lib/` mirrors Dash component template structure
- Backend: flat module layout inside `pivot_engine/pivot_engine/` with functional sub-packages (`backends/`, `cache/`, `planner/`, `runtime/`, `types/`)

## Where to Add New Code

**New UI feature in the grid (e.g. new toolbar button):**
- Add UI in `dash_tanstack_pivot/src/lib/components/DashTanstackPivot.react.js`
- If logic is self-contained and stateful, extract to a new hook in `dash_tanstack_pivot/src/lib/hooks/`
- Rebuild with `npm run build` from `dash_tanstack_pivot/`

**New filter type:**
- Add filter component: `dash_tanstack_pivot/src/lib/components/Filters/`
- Wire into `FilterPopover.js` and the main component's `handleHeaderFilter` logic

**New sidebar panel section:**
- Add component: `dash_tanstack_pivot/src/lib/components/Sidebar/`
- Register in the sidebar tab switcher inside `DashTanstackPivot.react.js`

**New backend query operation:**
- Add method to `TanStackPivotAdapter` in `pivot_engine/pivot_engine/tanstack_adapter.py`
- Route it through `PivotRuntimeService` in `pivot_engine/pivot_engine/runtime/service.py`
- Map new Dash props in `pivot_engine/pivot_engine/runtime/dash_callbacks.py`

**New Dash prop (backend → frontend):**
- Add to `DashTanstackPivot.propTypes` in `DashTanstackPivot.react.js`
- Add to auto-generated `DashTanstackPivot.py` (or regenerate via `npm run build:py`)
- Handle in the Dash callback output list in `dash_callbacks.py`

**New theme:**
- Add entry to `themes` object in `dash_tanstack_pivot/src/lib/utils/styles.js`
- No other changes required; the theme switcher reads `Object.keys(themes)` dynamically

**New test:**
- Backend unit tests: `pivot_engine/tests/test_*.py`
- Frontend contract / integration tests: `tests/test_frontend_contract.py`
