## Tech Debt (No Blockers)

### Issue 1 — Medium: Unguarded Module-Scope Import in Test File

**File:** `tests/test_frontend_contract.py` line 14
**Import:** `from dash_presentation.app import app`
**Problem:** `dash_presentation/app.py` executes `load_initial_data()` at module scope — generates 2,000,000 rows, loads into DuckDB, calls `materialized_hierarchy_manager.create_materialized_hierarchy()`. Runs on every pytest collection even for tests that never reference `app`. No `pytest.importorskip`, no `try/except`, no lazy fixture.
**Risk:** If any new dependency is added to `app.py` startup and is missing from the test environment, all 9 tests in the file become collection errors simultaneously — the exact failure mode Phase 1 spent effort fixing.
**Affected requirements:** BUG-03, BUG-04, QUAL-01
**Recommended fix:** Wrap the data generation in a `@pytest.fixture(scope="session")` and lazy-initialize via fixture injection rather than module-scope side effect.

---

### Issue 2 — Low: Unreachable `except Exception` Block

**File:** `dash_presentation/app.py` lines 308–314
**Problem:** Two consecutive `except Exception:` clauses inside `update_pivot_table`. Python matches the first; the second is dead code. The second block also returns a 6-tuple while the first returns a 7-tuple — a latent Dash callback arity mismatch if ever reordered during future editing.
**Affected requirements:** BUG-02
**Recommended fix:** Remove the second `except Exception` block.

---

### Issue 3 — Low: Frontend-Only Fixes Have No Automated Regression Tests

**Affected requirements:** BUG-08, BUG-09, BUG-10, BUG-13
**Problem:** The React cache invalidation (`useServerSideRowModel.js`), grouped-header sizing (`DashTanstackPivot.react.js`), header span widths, and context-menu clamping (`ContextMenu.js`) are verified only by webpack build success + phase VERIFICATION.md self-assessment. No `pytest` test exercises these code paths.
**Risk:** A future edit reverting these fixes would be invisible to the automated test suite.
**Recommended fix:** Address in Phase 7 (Code Quality Refactor) when Jest or Playwright testing infrastructure is introduced.
